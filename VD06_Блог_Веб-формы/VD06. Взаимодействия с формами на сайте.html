1. Создание форм в HTML

В уроке мы продолжим изучение веб-разработки и рассмотрим взаимодействие с формами на сайте. Формы являются важным элементом веб-страниц, позволяющим пользователям вводить и отправлять информацию.

Что будет на уроке:

  1. Узнаем, как создать форму на сайте.
  2. Научимся обрабатывать данные из формы на стороне сервера с использованием Flask.
  3. Разберемся, как вернуть результаты обработки данных обратно на клиентскую сторону.

🧠 Веб-форма — это аналог бумажной формы, которую заполняют, чтобы предоставить информацию. Она состоит из различных полей, списков и переключателей, позволяющих пользователю выбрать один или несколько элементов. Формы используются для получения сведений от посетителей сайта, например, для регистрации, входа в аккаунт, отправки заявок или отзывов.

Начнем с создания формы в HTML, а затем перейдем к работе с ней через Flask.

Сперва создаём отдельную страничку html и назовём её blog.html

  • Создадим HTML-страницу blog.html в PyCharm и начнем работу с формами.
  • В форме необходимо указать несколько обязательных атрибутов: action и method.
    • action указывает адрес, куда будут отправлены данные из формы.
    • method определяет способ отправки данных (GET или POST).

Основные теги для создания форм:

  1. <form> — основной тег для создания формы, который является контейнером для других элементов формы.
  2. <input> — тег для ввода данных. Включает текстовые поля, кнопки и радио-кнопки.
  3. <label> — тег для описания полей ввода.
  4. <textarea> — многострочное текстовое поле.
  5. <button> — тег для создания кнопок.

Код создания формы

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Title</title>
</head>
<body>
    <h1>Контактная форма</h1>
    #определяет форму с адресом для отправки данных и методом POST
    <form action="test" method="post">
        #создает метку для поля ввода имени
        <label for="name">Имя:</label>
        #оздает текстовое поле для ввода имени с обязательным заполнением
        <input type="text" id="name" name="name" required>
        <br>
        #создает метку для поля ввода электронной почты
        <label for="email">Электронная почта:</label>
        #создает поле для ввода электронной почты с обязательным заполнением
        <input type="email" id="email" name="email" required>
        <br>
        #создает кнопку для отправки формы
        <button type="submit">Отправить</button>
    </form>
</body>
</html>

Объяснение кода

  1. <form action="test" method="post"> — определяет форму с адресом для отправки данных из форма (пока поставим test и вернемся позже). Далее необходимо прописать метод: get — получение информаци, post — отправка информации.
  2. <label for="name">Имя:</label> — создает метку для поля ввода имени. Атрибут For указывает, к чему будет привязано наше название — поле должно совпадать с id поля, к которому идет привязка.
  3. <input type="text" id="name" name="name" required> — создает текстовое поле для ввода имени с обязательным заполнением (атрибут required). Атрибут name задает имя поля, которое будет потом отправляться на сервер, через это имя поля мы будем с ним работать и брать информацию.
  4. <br> — перенос строки.
  5. <label for="email">Электронная почта:</label> — создает метку для поля ввода электронной почты.
  6. <input type="email" id="email" name="email" required> — создает поле для ввода электронной почты с обязательным заполнением.
  7. <button type="submit">Отправить</button> — создает кнопку для отправки формы. Тип кнопки определяет действие кнопки.

2. Взаимодействие с формами через Flask

Создание структуры проекта Flask

Перед началом работы с формами через Flask необходимо создать правильную структуру проекта. В предыдущих проектах мы использовали упрощенную структуру, а сейчас создадим её по канонам, как указано в документации Flask.

Для начала взглянем на страницу с документацией Flask. Мы уже знаем, что такое модули и что такое пакеты. Flask-проекты создают именно как раз в виде пакета и за его пределами создают отдельный файл, через который запускается приложение. Сейчас мы сделаем что-то подобное, хоть и не такую сложную структуру, как в документации Flask.

Создание пакета проекта:

  • Создаем новый проект в PyCharm.
  • Создаем Python пакет: правой кнопкой мыши выбираем New — Python Package.
  • Даем название этому пакету: app. Внутри пакета уже присутствует файл init.py, который должен быть в проекте обязательно.

Создание структуры папок и файлов:

  • Внутри пакета app создаем папку templates для хранения HTML-шаблонов.
  • Внутри того же пакета app создаем файлы routes.py и __init__.py.
  • В корневом каталоге проекта создаем файл main.py.

Папку static мы создавать не будем, потому что стилизовать проект мы будем с помощью Bootstrap, поэтому static со стилями создавать необязательно.

Итак, у нас есть четыре основных файла: blog.html, init.py, routes.py и main.py. Начнём с ними работать.

Устанавливаем Flask через Python Package Manager, находим в строке поиска Flask и нажимаем Install.

Настройка Flask в __init__.py

В файле __init__.py настраиваем основные параметры Flask.

1. Импортируем и создаем экземпляр Flask:

from flask import Flask

#создаёт экземпляр класса Flask (переменную app)
app = Flask(__name__)

2. Настройка конфигурации:

  • Создаем секретный ключ для защиты данных:
app.config['SECRET_KEY'] = 'your_secret_key'

Данный ключ использует Flask для защиты данных сессий от различных атак, которым может подвергаться сайт. Здесь можно указать конкретный секретный ключ; если что-то разрабатывается в тестовом режиме, можно указать случайную фразу (чаще всего “you will never guess”).

3. Импортируем маршруты:

from app import routes

Эта строка нужна, чтобы из пакета app импортировать все routs. Внутри rout мы будем прописывать пути. Обычно мы их прописывали внутри файла main.py, а сейчас мы будем создавать максимально правильную структуру.


Создание HTML-шаблона blog.html

Здесь мы создадим форму, которая будет получать текст и создавать пост из этого текста. То есть, мы будем получать данные из формы и сразу же их далее использовать, возвращая пользователю, чтобы он мог увидеть пост, который он только что сделал.

Создание структуры HTML-документа:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <link rel="stylesheet" href="https://maxcdn.bootstrapcdn.com/bootstrap/4.5.2/css/bootstrap.min.css">
    <title>Простой блог</title>
</head>
<body>
#создает класс container
    <div class="container">
        <h1>Простой блог</h1>
        #создает форму с методом POST, чтобы отправлять данные из этой формы
        <form method="POST" action="/">
            <div class="form-group">
                <label for="title">Заголовок:</label>
                <input type="text" class="form-control" id="title" name="title" required>
            </div>
            <div class="form-group">
                <label for="content">Текст поста:</label>
                <textarea class="form-control" id="content" name="content" rows="3" required></textarea>
            </div>
            <button type="submit" class="btn btn-primary">Отправить пост</button>
        </form>
        <hr>
       #использует цикл for, чтобы иметь возможность публиковать посты
        <h2>Посты</h2>
        {% for post in posts %}
       #создает отдельный div, внутри которого записываются заголовки и текст постов
        <div class="post">
            <h4>{{ post.title }}</h4>
            <p>{{ post.content }}</p>
        </div>
        {% endfor %}
    </div>
    #подключает Bootstrap для стилизации
    <script src="https://ajax.googleapis.com/ajax/libs/jquery/3.5.1/jquery.min.js"></script>
    <script src="https://maxcdn.bootstrapcdn.com/bootstrap/4.5.2/js/bootstrap.min.js"></script>
</body>
</html>

Ещё раз вспомним, как пользоваться Bootstrap: на сайте выбираем метод подключения via CDN, копируем первую ссылку и вставляем в код наверху, над тегом head, затем копируем вторую ссылку с сайта и вставляем её внизу, перед закрывающим тегом body.

Далее обращаемся к Нейрокоту, чтобы он помог стилизовать проект с помощью Bootstrap. Для этого копируем из кода всё, что находится внутри контейнера div и передаём эту информацию Нейрокоту. Просим его переписать данный код, подключив стили Bootstrap и сделать красивый сайт.

Копируем ответ Нейрокота и вставляем в файл blog.html. Теперь страница в браузере выглядит гораздо аккуратнее и удобнее.

2. Взаимодействие с формами через Flask

В предыдущей части урока мы подготовили файлы init.py и blog.html. Теперь необходимо прописать логику того, как мы будем брать данные, которые вводятся в форму, и что с этими данными будет происходить дальше.

Импорт необходимых библиотек

Теперь мы будем работать в файле routes.py.

Импортируем из Flask:

  • render_template: для рендеринга HTML-шаблонов.
  • request: для работы с данными HTTP-запросов.
  • redirect и url_for: для перенаправления пользователей и создания URL по имени функции.

Для этого прописываем в файле routes.py:

from render_template, request, redirect, url_for

Импорт переменной app из модуля app, чтобы использовать экземпляр класса Flask. Для этого прописываем:

from app import app

Работа с данными формы

Инициализируем пустой список posts для хранения постов:

posts - []

Создаем маршрут: используем @app.route("/", methods=["GET", "POST"]) для определения главного маршрута с методами GET и POST:

@app.route("/", methods=["GET", "POST"])

Таким образом, здесь мы указали, что данный маршрут поддерживает и GET-методы, и POST-методы.

Создание функции

def index():
#использует метод POST, так как информация будет отправляться. Request method сравнивает данные с HTTP-запросом.
    if request.method == 'POST':
    #функция request.form извлекает значение из соответствующих полей
        title = request.form.get('title')
        content = request.form.get('content')
        #создаёт условие для проверки наличия данных в полях title и content
        if title and content:
        posts.append({'title': title, 'content': content})
        #использует для обновления страницы и предотвращения повторной отправки формы.
        return redirect(url_for('index'))
        #возвращает отрендеренный шаблон с переданными данными постов
    return render_template('blog.html', posts=posts)

Таким образом, цикл for будет перебирать список словарей, и каждый отдельный словарь с заголовком и контентом будет сохраняться в переменную posts. Также мы будем извлекать оттуда что-то конкретное (заголовок или контент) и всё это прописывать.

Настройка запуска приложения

В файле main.py импортируем app из модуля app:

from app import app

Используем условие для проверки, что скрипт запускается напрямую:

if __name__ == "__main__":
    app.run(debug=True)

Теперь мы можем протестировать приложение. Запускаем сервер и переходим по ссылке для тестирования. Вводим заголовок и текст поста, отправляем форму и проверяем отображение поста. Сайт работает исправно.
