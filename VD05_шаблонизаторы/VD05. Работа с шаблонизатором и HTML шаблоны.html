Мы продолжаем изучение веб-разработки, наша тема сегодня — работа с шаблонизатором и HTML-шаблонами.

План урока:

  • узнаем, что такое Jinja;
  • разберём основные концепции Jinja: из чего состоит шаблон, как выполнить наследование шаблонов, какие полезные свойства есть у шаблонов;
  • попробуем использовать Jinja в проекте.

🧠 Jinja — это один из самых популярных шаблонизаторов для языка Python. Он используется для генерации HTML, XML или других текстовых форматов из шаблонов. Jinja позволяет отделить логику приложения от его представления, что делает код более организованным и читаемым. 

Jinja — это небольшой набор инструментов, который помогает автоматизировать создание HTML-шаблонов. HTML-шаблоны часто используются, потому что все страницы в интернете имеют примерно одну структуру: сверху меню, ниже контент, ещё ниже футер (подвал). На многостраничном сайте страницы будут ещё более однотипными из-за дизайна. В них меняться будет только контент.

С помощью шаблона мы можем выделить меню и футер, а изменять только контент. Нам не нужно будет заново разрабатывать меню и футер для каждой страницы — они будут подключаться автоматически.

🧠 Шаблонизатор — это ПО, позволяющее использовать HTML-шаблоны для генерации HTML-страниц. 

Как мы можем использовать Jinja:

Есть код с прошлого урока для Flask-проекта.

from flask import Flask, render_template

app = Flask(__name__)

@app.route("/")
def films():
    return render_template("index.html")

@app.route("/person/")
def person():
    return render_template("main.html")

if __name__ == "__main__"
    app.run()

При установке Flask Jinja устанавливается автоматически. На всякий случай проверяем:

  1. В правом верхнем углу нажимаем на кнопку настроек (шестерёнка) — Preferences...
  2. В левой секции открывшегося окна выбираем Project: [Название проекта].
  3. В рабочей области окошка выбираем Python Interpreter. Видим список всех установленных модулей — находим Jinja.
  4. Нажимаем на кнопку OK.

Если что-то пошло не так, устанавливаем Jinja отдельно:

  1. Переходим в Python Packages.
  2. Находим Jinja2.
  3. Нажимаем на кнопку Install package.

В Jinja есть переменные, которые мы можем использовать, чтобы вставлять информацию в шаблон. В переменные мы можем разными способами вставлять разную информацию.

Функция render_template нужна для отрисовки шаблона. Помимо этого, эта функция может принимать неограниченное количество именованных аргументов. Эти аргументы функция будет пробрасывать в HTML-шаблон.


1. Добавляем функцию:

from flask import Flask, render_template

app = Flask(__name__)

@app.route("/")
def films():
    return render_template("index.html")
    
@app.route("/shablon/")
def films2():
    return render_template("index.html")

@app.route("/person/")
def person():
    return render_template("main.html")

if __name__ == "__main__"
    app.run()

2. Переходим в документ index.html и создаём переменную шаблонизатора:

# Было вот так:
<title>Фильмы</title>

# Делаем вот так:
<title>{{caption}}</title>

{{}} — это аналог функции print в Python. Всё будет выводиться в название вкладки.

caption — это переменная. Через Flask мы можем вставить в неё информацию.

3. Переходим в документ main.py и заполняем переменную:

from flask import Flask, render_template

app = Flask(__name__)

@app.route("/")
def films():
    return render_template("index.html", caption="Фильмы про Гарри")
    
@app.route("/shablon/")
def films2():
    return render_template("index.html", caption="Гарри Поттер")

@app.route("/person/")
def person():
    return render_template("main.html")

if __name__ == "__main__"
    app.run()

4. Запускаем main.py и проверяем обе страницы.

Теперь мы имеем два разных заголовка для одной и той же страницы. Это можно использовать при создании двух однотипных сайтов.

Такие переменные можно создавать по всему шаблону, в любом теге HTML-файла, не обязательно только в названии. Количество таких переменных не ограничено.

Но создание большого количества переменных сделает строчки кода длинными и неудобными. Поэтому все переменные рекомендуется сохранять в словарь и распаковывать его.

Используем словарь:


1. Переходим в документ main.py и создаём словарь — разный для двух страниц:

3. Наследование шаблонов

Принцип DRY (Don't repeat yourself, “не повторяйся”) предостерегает нас от дублирования кода или логики программы.

Это значит, что вместо дублирования и вставки кусков кода в другие места нам лучше вынести повторяющиеся части в функции, модули, классы или в шаблон. Это экономит время, потому что для изменения повторяющейся части достаточно будет изменить его в одном месте. Также легче поддерживать и тестировать код, он легче читается, более структурированный.


У нас есть две страницы: main.html и index.html. Они отличаются только контентом, у них одинаковая навигационная панель. Мы можем сделать для них одинаковый футер.


  1. Переходим в Нейрокота и отправляем промпт:

✍🏻 Напиши простой футер для сайта про фильмы с использованием bootstrap

2. Копируем футер. Вставляем его в оба HTML-файла в конце кода, после последнего </div>.

3. Запускаем сайт.


При запуске через HTML-файл переменные будут выглядеть как {{link}}, в них не подставляются значения. Это нормально, всё будет работать при запуске через Flask.

Настраиваем шаблон. Для этого используем отдельный файл — shablon.html:

  1. Копируем всю повторяющуюся часть (меню и футер) из main.html или index.html.

2. Вставляем в shablon.html.

3. Создаём блок внутри тега <title>. Здесь будет находиться название.

<title>
    {% block title %}
    
    {% endblock %}
</title>

4. Создаём блок для контента. Делаем это после меню, перед футером.

🧠В блоки будет вставляться информация, которая не повторяетс

{% block content %}
    
{% endblock %}

5. Из файлов main.html и index.html удаляем все повторяющиеся части кода.


6. В начале обоих файлов пишем указание на использование шаблона:

{% extends 'shablon.html' %}

{% block title %}
    Фильмы про Гарри Поттера
    # В файле main.html пишем заголовок: Персонажи вселенной Гарри Поттера
{% endblock %}

7. Указываем использование шаблона для контента:

{% block content %}
    # здесь уже написанный контент, который не повторяется
{% endblock %}

8. Редактируем код в main.py:

from flask import Flask, render_template

app = Flask(__name__)

@app.route("/")
def films():
    context = {
        "link": "Перейти в кинотеатр"
    }
    return render_template("index.html", **context)

@app.route("/person/")
def person():
    context = {
        "link": "Перейти в кинотеатр"
    }
    return render_template("main.html", **context)

if __name__ == "__main__"
    app.run()

9. Запускаем сайт. Меню и футер будут показаны, хотя на страницах нет кода для меню и футера.
